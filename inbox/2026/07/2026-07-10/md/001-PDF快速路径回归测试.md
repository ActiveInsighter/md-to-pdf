# PDF 快速路径回归测试

> 本文档用于验证优化后的 Markdown 到 PDF 快速导出流程，包括一次性提交、自动构建、状态轮询、质量检查和产物下载。

## 1. 测试目标

本次测试只新增临时 `inbox` 任务，不修改渲染器、主题、工作流或依赖。预期流程如下：

```text
准备 Markdown 与 manifest
→ 一次提交进入 main
→ GitHub Actions 自动构建
→ 自动生成 PDF、HTML、预览图和质量报告
→ 发布到 output 分支
→ 消费临时 inbox 任务
```

## 5. 代码块检查

```typescript
interface BuildResult {
  status: "success" | "failure";
  pdf: string;
  pages: number;
}

function isValid(result: BuildResult): boolean {
  return result.status === "success" && result.pages > 0;
}
```

## 6. 验收标准

1. 构建状态为 `success`。
2. PDF、HTML、预览图和质量报告均已生成。
3. 未检测到 KaTeX 错误、图片缺失、空白页或横向溢出。
4. 页脚页码使用项目当前的优化样式。
5. 构建成功后临时 `inbox` 任务被自动消费。
