# 引用与表格样式验证

这份文档用于验证 `chatgpt-light` 主题下的引用块、Markdown 表格、中文换行和列分割效果。

> 考点追踪：引用块应保持浅灰背景、清晰左侧竖线、紧凑内边距，并且不应显得过重或过宽。

## 1. Cache 映射方式表格

| 标记 | Cache 行号 | 块内地址 |
| --- | --- | --- |
| 主存块号的高 \( t \) 位，用于判断该 Cache 行当前保存的是哪一个主存块 | 主存块号的低 \( c \) 位，用于确定映射到哪一行 | 块内偏移地址，用于确定块中具体字节或字的位置 |

## 2. 对比说明

| 映射方式 | 优点 | 缺点 |
| --- | --- | --- |
| 直接映射 | 实现简单，访问速度快，只需要比较一个标记字段 | 冲突概率高，空间利用率较低 |
| 全相联映射 | 主存块可以装入任意 Cache 行，冲突概率低，命中率较高 | 标记比较速度慢，实现成本较高，通常需要相联存储器 |
| 组相联映射 | 兼顾直接映射和全相联映射，在速度、成本和命中率之间折中 | 组内仍需比较多个标记，实现复杂度高于直接映射 |

## 3. 长内容换行测试

| 项目 | 说明 |
| --- | --- |
| 换行策略 | 表格中的中文长句应按正常中文阅读节奏换行，不应在每个字符之间生硬断开，也不应因为单元格内容较长而撑破页面。 |
| 列分割 | 列之间应有浅色分割线，视觉上能区分列，但不能像粗边框一样抢占正文注意力。 |
| 宽度策略 | 两列和三列表格不应被强制拉得过宽；四列及以上表格应优先使用整页宽度并稳定换行。 |
